# DS18B20温度报警器单元测试

## 概述

本目录包含针对DS18B20温度报警器项目的单元测试。测试覆盖了以下模块：
1. DS18B20温度传感器驱动
2. 数码管显示模块
3. 主函数逻辑

## 测试环境要求

- GCC编译器 (MinGW for Windows 或 gcc for Linux)
- Make工具
- Windows/Linux命令行环境

## 快速开始

### Windows用户
1. 双击 `run_tests.bat` 文件
2. 或打开命令行，进入tests目录，运行:
   ```
   run_tests.bat
   ```

### Linux/Mac用户
1. 打开终端，进入tests目录
2. 运行:
   ```
   make all-test
   ```

### 手动步骤
1. 编译测试程序:
   ```
   make all
   ```
2. 运行测试:
   ```
   ./run_tests.exe
   ```

## 测试框架

测试使用自定义的轻量级测试框架 (`test_framework.h/c`)，提供以下功能:
- 测试断言宏
- 测试统计
- 模拟硬件寄存器
- 模拟延时函数

## 模拟策略

由于测试在PC上运行，而非实际硬件，我们使用以下模拟策略:

### 硬件寄存器模拟
- 所有特殊功能寄存器(P3, P6, P7等)定义为普通变量
- IO引脚状态通过变量模拟

### 延时函数模拟
- `Delay_ms()` 和 `Delay_us()` 函数被模拟为空操作
- 仅验证函数被正确调用

### DS18B20设备模拟
- 设备存在脉冲响应被模拟
- 温度数据读取被模拟
- 通信时序被简化

## 覆盖率目标

测试设计覆盖以下关键路径:
- 正常温度读取和显示流程
- 边界温度值处理 (0°C, 125°C, 负温度)

## 故障排除

### 常见问题

1. **编译错误: 找不到头文件**
   - 确保在tests目录中运行
   - 检查`../`目录中是否存在相应的头文件

2. **链接错误: 未定义引用**
   - 测试是独立的，不链接实际硬件代码
   - 所有函数都在测试文件中实现或模拟

3. **测试失败: 模拟函数不匹配**
   - 检查模拟函数与真实硬件的差异
   - 可能需要更新模拟逻辑

### 添加新测试

1. 创建新的测试文件 `test_模块名.c`
2. 在测试文件中包含 `test_framework.h`
3. 实现测试函数
4. 在 `test_main.c` 中调用新测试
5. 更新Makefile（如果需要）
